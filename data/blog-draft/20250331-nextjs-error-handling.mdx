A few issues I've noticed in the <a href="https://nextjs.org/docs/app/getting-started/error-handling">Next.js official documentation on error handling</a>
that make it not really suitable for a production-grade application:

### Log on the client side

Why would you want to log to a reporting service from the client-side component?

```tsx
'use client' // Error boundaries must be Client Components

import { useEffect } from 'react'

export default function Error({
  error,
  reset,
}: {
  error: Error & { digest?: string }
  reset: () => void
}) {
  useEffect(() => {
    // Log the error to an error reporting service
    console.error(error)
  }, [error])

  ...
}
```
